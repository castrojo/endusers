Cloud native runs on the organizations that use it. For every project in the CNCF
landscape there are teams running it in production — debugging it at 3 AM, scaling it
past what anyone thought reasonable, and quietly figuring out the patterns the rest of
us end up adopting. This site is for them.

**endusers.cncf.io** is the new home of the [CNCF End User
Community](https://www.cncf.io/enduser/): the practitioners, architects, and
organizations adopting cloud native technologies to build things that matter.

<!-- truncate -->

## What lives here

- **[Reference Architectures](/architectures/)** — real architectures from real
  organizations, reviewed through the [End User TAB's submission
  process](https://github.com/cncf/tab/blob/main/process/reference-architectures.md) and
  imported directly from the [CNCF architecture
  repository](https://github.com/cncf/architecture). Not whiteboard idealizations — the
  teams, trade-offs, and lessons behind production systems at organizations like Adobe,
  CERN, and Swisscom.
- **[Community](/community/)** — the [End User Technical Advisory Board
  (TAB)](https://github.com/cncf/tab), industry-focused End User Groups, meeting
  schedules, and every channel where end users compare notes.
- **[End User Awards](/awards)** — the organizations CNCF has recognized for running
  cloud native at its best, from Bloomberg in 2018 to ANT Group in 2025.
- **[Metrics](/metrics/)** — a transparent snapshot of ecosystem and end-user activity,
  generated from authoritative public sources. If we can't source a number, we leave it
  out.
- **[Events](/events/)** — where the community gathers at KubeCon + CloudNativeCon,
  including the End User Summit and End User Reception.

## Who this is for

If you operate cloud native systems in production — whether you're evaluating your
first CNCF project or running a thousand clusters — this site is built for you. You
don't need to be a CNCF member organization to learn from the architectures, join the
Slack conversations, or show up at community meetings.

## How to get involved

- Join the `#enduser` channel in [CNCF Slack](https://slack.cncf.io/)
- Subscribe to the [End User mailing list](https://lists.cncf.io/g/cncf-enduser)
- Attend the [TAB public meeting](https://github.com/cncf/tab#meetings), held the third
  Monday of every month
- Share your architecture through the [reference architecture submission
  process](https://github.com/cncf/tab/blob/main/process/reference-architectures.md)
- Bring your organization into the [End User Community](https://www.cncf.io/enduser/)

The best thing about the cloud native community has always been how freely it shares
what it learns. This site exists to make that sharing easier. Welcome.
